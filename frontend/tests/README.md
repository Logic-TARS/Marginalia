# Reader Boundary Tests

## Running Tests
```bash
cd frontend
npm install
npx playwright install chromium  # or use system Chrome via channel: 'chrome'
npx playwright test
```

## Test Tags
- `@smoke` — verifies the EPUB fixture loads correctly
- `@boundary.forward` — cross-section forward navigation lands on page 1 of new chapter
- `@boundary.backward` — cross-section backward navigation lands on last page of prior chapter
- `@boundary.intra` — same-section navigation advances exactly +1/-1 page

## Fixture
The test fixture (`tests/fixtures/multichapter.epub`) is generated by `tests/fixtures/build-fixture.mjs`.
It contains 3 chapters, each ≥220 paragraphs (~50KB), rendered at 1000×700 viewport.
**Do NOT regenerate the fixture unless the content changes** — it is committed as a frozen binary.
The generator uses fixed ZIP timestamps (1980-01-01) for byte-reproducible output.

## Escalation
If the wrap-guard in `navigatePage` does NOT eliminate the cross-section page skip
(after 3 consecutive Playwright runs where `displayed.page !== 1` for forward cross-section),
escalate to patching `frontend/epub.min.js` with an offset-annotated patch.
See `.omo/plans/fix-chapter-boundary-page-skip.md` for details.
