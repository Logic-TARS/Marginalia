## 问题摘要

阅读器通过左右按钮翻页时存在以下异常：

- 第一次点击下一页后，正文位置已变化，但页码仍显示第 1 页。
- 章内页码看起来少走一页，造成翻页跳页的观感。
- 测试书籍的第一章显示共有 45 页，但会在第 43 页后提前进入第二章。
- 跨章后页码直接从 `43 / 45` 变为下一章的 `1 / 45`。

问题同时影响正文翻页反馈和页码可信度，但不涉及 EPUB 文件损坏、后端数据或阅读进度数据库。

## 根因分析

### 样式选择器冲突

应用使用以下宿主元素承载 `epub.js`：

```html
<div class="epub-container" id="epub-container">
```

`epub.js` 在该元素内动态生成自己的 `.epub-container`。应用原有的通用 CSS 选择器因此同时命中了宿主容器和库生成的内部容器，给内部容器重复添加了边框、阴影和 flex 属性。

### 运行时证据

修复前采集到的容器尺寸为：

| 容器 | 外部宽度 | `clientWidth` | 边框 |
| --- | ---: | ---: | --- |
| 应用宿主容器 | 616px | 614px | 1px |
| `epub.js` 内部容器 | 614px | 612px | 1px |

`epub.js` 按内部 612px 宽度生成分页，但默认视图管理器使用外层边界判断是否还能显示下一页。2px 的尺寸偏差导致分页映射和翻页边界不一致：

- 第一次翻页后 CFI 已前进，显示页码仍为第 1 页。
- 第一章实际继续翻页时，页码依次显示到 `43 / 45`，随后提前切换章节。

通过浏览器运行时临时移除内部容器的应用边框后，内部容器 `clientWidth` 恢复为 614px，翻页序列立即恢复为：

```text
1 -> 2 -> 3 -> ... -> 44 -> 45 -> 下一章 1
```

因此根因确定为应用 CSS 类名与 `epub.js` 内部类名冲突，而不是按钮重复绑定、CFI 丢失或章节导航状态机故障。

## 修复内容

### 隔离阅读器宿主样式

- 将宿主视觉类从 `.epub-container` 改为 `.reader-epub-host`。
- 同步调整 iframe 和移动端 CSS 选择器。
- 保留稳定的 `#epub-container` ID，现有 JavaScript DOM 引用和自动化定位无需改变。
- `epub.js` 动态生成的 `.epub-container` 不再继承应用的边框、阴影和 flex 样式。

涉及文件：

- `frontend/index.html`
- `frontend/style.css`

### 更新 PWA 缓存

Service Worker 采用 app shell 缓存优先策略。为避免已安装 PWA 继续使用旧版 CSS，缓存名称从：

```text
marginalia-v4
```

升级为：

```text
marginalia-v5
```

涉及文件：

- `frontend/sw.js`

### 强化回归测试

更新 `frontend/tests/reader-boundary.spec.js`，新增或收紧以下断言：

- `epub.js` 内部容器边框必须为 `0px`。
- 内部容器 `clientWidth` 必须等于宿主内容宽度。
- 第一次点击下一页必须从第 1 页准确进入第 2 页。
- 章内每次点击必须严格增加一页。
- 正向跨章前必须完整到达 `N / N`，下一次点击进入相邻章节第 1 页。
- 反向跨章必须落在上一章节的 `N / N`。
- 480px 窄屏视口下同样保持一次点击只翻一页。

本次修复没有修改 `frontend/app.js` 的翻页状态机。

## 风险与后续注意事项

- 可重排 EPUB 的总页数会随视口尺寸和字体大小变化，这是正常行为；同一布局下翻页必须保持连续。
- 后续新增阅读器样式时，应避免使用第三方库可能生成的通用类名作为应用宿主类。
- 若再次出现页码与正文位置不同步，应先比较宿主容器、库内部容器和分页布局的实际宽度，再检查导航状态机。
- PWA 客户端更新后会清理旧缓存；已打开的旧页面可能需要刷新一次才能加载新 app shell。

## 回滚说明

如需回滚，应同时处理以下内容：

1. 恢复 `frontend/index.html` 和 `frontend/style.css` 中对应的宿主类名。
2. 调整 `frontend/sw.js` 缓存版本，确保客户端获取匹配的 HTML 和 CSS。
3. 保留严格翻页测试，用于确认回滚没有重新引入容器尺寸偏差。

不应只回滚 HTML 或 CSS 中的一侧，否则会导致阅读器宿主样式丢失或再次污染 `epub.js` 内部容器。
